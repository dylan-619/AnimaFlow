## 用户需求

针对 AnimaFlow 项目分析报告中"第一阶段：快速提升（1-2周）"的5个核心任务制定详细整改计划：

| 序号 | 任务 | 预期效果 | 工作量 |
| --- | --- | --- | --- |
| 1 | 优化分镜生成提示词 | 提高镜头拆分质量 | 2天 |
| 2 | 优化视频提示词生成 | 提高视频生成成功率30% | 2天 |
| 3 | 引入角色资产参考机制 | 提高角色一致性40% | 3天 |
| 4 | 自动拆解复杂镜头 | 提高视频连贯性 | 2天 |
| 5 | 失败重试机制 | 提高成功率 | 1天 |


## 当前状态分析

**已有能力**：

- 分镜生成已支持 `shotAction`(动作序列)、`shotDuration`(时长)、`cameraMovement`(运镜) 字段
- 分镜图生成已支持 `referenceImages` 参考图参数
- 视频提示词润色接口已使用 `shotAction` 替代 `shotPrompt`
- 资产关联通过 `relatedAssets` 字段实现

**核心问题**：

1. **提示词模板优化空间**：当前模板对时序分解、动作拆分指导不够具体
2. **资产参考机制未完全激活**：分镜图生成时已有参考图逻辑，但资产匹配精度可优化
3. **复杂镜头未拆解**：`shotAction` 包含多个动作时未自动拆分为多个分镜
4. **任务重试缺失**：`taskRunner.ts` 中任务失败后直接标记 failed，无重试机制

## 技术栈

- **后端框架**: Node.js + Express 5 + TypeScript
- **数据库**: SQLite (better-sqlite3)
- **AI 文本**: DeepSeek / OpenAI GPT
- **AI 图像**: 豆包 Seedream (支持参考图 `image` 参数)
- **AI 视频**: 豆包 Seedance

## 实现方案

### 任务1：优化分镜生成提示词（2天）

**目标**: 提高镜头拆分质量，生成更精准的 `shotAction` 和 `shotDuration`

**修改文件**:

- `docs/05_分镜生成_storyboard.md` - 优化提示词模板
- `server/src/db/init.ts` - 更新数据库中的默认模板

**优化要点**:

1. 增加动作复杂度判断规则（简单动作2-3秒，复杂动作拆解）
2. 增加 `shotAction` 时序分解示例
3. 量化镜头时长估算规则
4. 增加镜头衔接状态标注（`prev_state`, `next_state`）

### 任务2：优化视频提示词生成（2天）

**目标**: 提高视频生成成功率30%，生成更精准的英文提示词

**修改文件**:

- `docs/10_视频提示词生成_video_prompt.md` - 优化提示词模板
- `server/src/services/videoService.ts` - 增强 `polishPrompt` 接口
- `server/src/db/init.ts` - 更新数据库中的默认模板

**优化要点**:

1. 增加动作时序分解逻辑（0-1秒做什么，1-3秒做什么）
2. 增加情绪指导字段（emotion、speed、intensity）
3. 增加首帧状态和尾帧状态描述
4. 优化运镜指令与动作的匹配

### 任务3：引入角色资产参考机制（3天）

**目标**: 提高角色一致性40%，确保同一角色在不同镜头中外观一致

**修改文件**:

- `server/src/services/storyboardService.ts` - 增强资产匹配逻辑
- `server/src/services/assetsService.ts` - 增加资产特征提取
- `server/src/ai/image.ts` - 优化参考图权重参数

**优化要点**:

1. 增加资产匹配精度（使用向量相似度或关键词权重）
2. 增加角色主参考图机制（`isMasterReference` 字段已存在，需激活使用）
3. 增加参考图权重配置（`strength` 参数）
4. 增加资产特征一致性检查

### 任务4：自动拆解复杂镜头（2天）

**目标**: 提高视频连贯性，将复杂动作拆解为多个简单分镜

**修改文件**:

- `server/src/services/storyboardService.ts` - 增加 `decomposeComplexShot` 函数
- `docs/05_分镜生成_storyboard.md` - 增加拆解规则说明

**优化要点**:

1. 定义复杂动作判定规则（关键词：然后、之后、同时、一边、并）
2. 实现动作拆解算法（按时间节点拆分）
3. 自动生成衔接镜头（状态过渡）
4. 保持 `relatedAssets` 关联一致性

### 任务5：失败重试机制（1天）

**目标**: 提高生成成功率，减少因网络抖动导致的任务失败

**修改文件**:

- `server/src/task/taskRunner.ts` - 增加重试逻辑
- `server/src/types/index.ts` - 增加 `retryCount` 字段定义
- `server/src/db/init.ts` - 增加数据库字段迁移

**优化要点**:

1. 增加最大重试次数配置（默认3次）
2. 增加指数退避等待（1s, 2s, 4s）
3. 区分可重试错误类型（网络错误、超时 vs 业务错误）
4. 记录重试次数和错误历史

  maxDelay: number;             // 最大延迟(ms)，默认16000

interface DecomposedShot {
  originalShotId: number;
  subShots: {
    shotIndex: number;
    action: string;
    duration: number;
    startState: string;
    endState: string;
  }[];
}

// 任务3：资产参考配置
interface AssetReferenceConfig {
  characterWeight: number;      // 角色参考图权重，默认0.7
  sceneWeight: number;          // 场景参考图权重，默认0.5
  matchThreshold: number;       // 匹配阈值，默认0.6
}
