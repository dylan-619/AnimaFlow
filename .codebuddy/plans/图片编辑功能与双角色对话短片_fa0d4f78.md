## 用户需求

### 需求一：图片编辑功能（优先）

为资产和分镜图片添加图生图编辑功能，支持三种编辑模式：

- **局部修改**：调整表情、姿势、细节等
- **风格迁移**：改变美术风格（如写实转动漫）
- **画面扩展/裁剪**：扩展画面边界或裁剪画面

### 需求二：现代都市测试短片

生成一个约1分钟的测试短片：

- 现代都市题材
- 两个角色的对话
- 包含旁白（旁白需要生成TTS音频 + 在对话列表中显示）

## 产品概述

1. **图片编辑模块**：在现有的资产/分镜图片生成弹窗中增加编辑模式选择，用户可基于现有图片进行图生图修改，提高创作灵活性
2. **测试短片**：参考极简测试短片格式，创作一个现代都市题材的对话短片剧本，用于验证双角色对话+旁白的完整流程

## 核心功能

- 编辑模式切换（新建/局部修改/风格迁移/画面扩展）
- 编辑强度滑块控制
- 编辑描述输入框
- 资产图片编辑 API
- 分镜图片编辑 API
- 现代都市测试短片剧本生成

## 技术栈

- **前端框架**: Vue 3 + TypeScript + Element Plus
- **后端框架**: Node.js + Express 5 + TypeScript
- **AI 图像**: 豆包 Seedream（已支持 `image` 参考图参数）
- **AI 文本**: DeepSeek / OpenAI GPT
- **数据库**: SQLite (better-sqlite3)

## 关键代码结构

```typescript
// 编辑模式类型定义
type EditMode = 'create' | 'local_edit' | 'style_transfer' | 'expand';

// 图片编辑请求参数
interface ImageEditRequest {
  sourceId: number;           // 资产ID或分镜ID
  sourceType: 'asset' | 'storyboard';
  mode: EditMode;
  sourceImage: string;        // 源图片路径
  editPrompt: string;         // 编辑描述
  strength: number;           // 编辑强度 0.3-0.8
}

// 图片编辑API调用
async function editImage(
  sourceImage: string,
  editPrompt: string,
  options: { mode: EditMode; strength: number }
): Promise<ImageResult[]> {
  // 根据编辑模式生成增强提示词
  const enhancedPrompt = generateEditPrompt(editPrompt, options.mode, options.strength);
  // 调用豆包图片生成API，传入参考图
  return imageGenerate(enhancedPrompt, { referenceImages: [sourceImage] });
}
```

## 设计概述

在现有的图片生成弹窗基础上，增加编辑模式选择和编辑控制面板，保持与现有UI风格一致。

## 编辑模式UI设计

### 模式选择区

- 使用 Radio Button Group 展示四种模式：新建、局部修改、风格迁移、画面扩展
- 选中编辑模式时，显示源图片选择区和编辑控制面板

### 编辑控制面板

- **源图片选择**：从历史图片网格中选择或上传新图片
- **编辑强度滑块**：0.3（轻度）到 0.8（大幅度），默认 0.5
- **编辑描述输入框**：文本域，描述想要的修改

### 视觉风格

- 保持现有弹窗的卡片式布局
- 使用 Element Plus 的 Radio Button 和 Slider 组件
- 编辑模式下源图片显示为半透明遮罩 + 编辑图标
