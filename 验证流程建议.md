# 极简测试短片验证流程建议

## 验证目标
使用 `极简测试短片.txt` 验证以下新增功能:
1. 视频比例自动适配 (9:16竖屏)
2. 角色/场景/道具一致性控制
3. 批量生成分镜图功能
4. 质量评估系统
5. 动作复杂度拆解规则

## 验证步骤

### 第一步: 项目初始化
```
POST /api/project/create
{
  "name": "极简分镜验证",
  "artStyle": "赛博朋克动漫",
  "videoRatio": "9:16",
  "styleGuide": "冷色调为主，高反差赛璐璃渲染。强调雨点和积水反光。多用特写和半身景别，避免复杂的全身大幅度运动，突出静谧感。"
}
```
**验证点**: 
- ✅ 项目创建成功
- ✅ videoRatio 字段正确保存为 "9:16"

---

### 第二步: 导入剧本
将测试文件内容作为小说导入:
```
POST /api/novel/create
{
  "projectId": "<项目ID>",
  "chapter": "夜雨寻踪",
  "chapterData": "<测试文件正文>"
}
```

---

### 第三步: 大纲生成
```
POST /api/outline/generate
{
  "projectId": "<项目ID>",
  "novelIds": ["<小说ID>"]
}
```
**验证点**:
- ✅ 大纲生成符合30秒短片要求
- ✅ 提取了角色(猎人K)、场景(霓虹窄巷)、道具(机械怀表)

---

### 第四步: 资产提取与生成
```
POST /api/assets/extract
{
  "projectId": "<项目ID>",
  "outlineId": "<大纲ID>"
}
```

**关键验证**: 资产图生成
```
POST /api/assets/generateImages
{
  "projectId": "<项目ID>"
}
```
**验证点**:
- ✅ 角色图生成正确(猎人K四视图)
- ✅ 场景图生成正确(霓虹窄巷氛围)
- ✅ 道具图生成正确(机械怀表)
- ✅ 生成的图片尺寸符合视频比例 (720x1280)

---

### 第五步: 分镜生成 (核心验证)
```
POST /api/storyboard/generate
{
  "projectId": "<项目ID>",
  "scriptId": "<剧本ID>"
}
```

**验证点 (重点)**:
- ✅ **动作复杂度拆解**: 检查分镜是否按新规则拆分复杂动作
  - 例如: "缓缓抬起右手...微微侧过头" 是否拆分为多个镜头?
- ✅ **时序校验**: 每个镜头的 shotDuration 是否合理
  - 预估动作时长 vs 实际时长
- ✅ **运镜指令**: cameraMovement 是否符合景别
  - 特写 → 静止/微推
  - 中景 → 推镜头/拉镜头
- ✅ **relatedAssets**: 是否正确关联了猎人K、霓虹窄巷、机械怀表

---

### 第六步: 分镜图生成 (批量验证)
```
POST /api/batch/storyboardImages
{
  "projectId": "<项目ID>",
  "concurrency": 3,
  "skipExisting": false
}
```

**验证点 (核心)**:
- ✅ **视频比例适配**: 检查生成的图片尺寸是否为 720x1280 (9:16)
- ✅ **参考图注入**: 检查日志中是否有 `参考图注入: X张, 权重=0.8`
- ✅ **一致性模式**: 检查日志中是否有 `consistencyMode: character-focused`
- ✅ **批量并行**: 观察是否按 concurrency=3 并行生成
- ✅ **视觉一致性**: 
  - 猎人K在不同镜头中是否外观一致?
  - 霓虹窄巷的氛围是否统一?
  - 机械怀表的细节是否保持?

---

### 第七步: 质量评估 (新功能验证)
```
POST /api/quality/batchAssess
{
  "projectId": "<项目ID>"
}
```

**验证点**:
- ✅ **清晰度评分**: clarity 是否合理 (文件大小检测)
- ✅ **一致性评分**: consistency 是否反映角色/场景一致性
- ✅ **美观度评分**: aesthetics 是否合理
- ✅ **提示词遵循度**: promptAdherence 是否合理
- ✅ **改进建议**: recommendations 是否具体可操作
- ✅ **需审核标记**: needsReview 是否正确标记低质量分镜

---

### 第八步: 视频提示词生成 (新规则验证)
```
POST /api/video/generatePrompts
{
  "projectId": "<项目ID>",
  "storyboardIds": ["<分镜ID数组>"]
}
```

**验证点 (文档增强验证)**:
- ✅ **动作速度控制词**: 检查英文提示词中是否包含
  - "slowly" (缓缓)
  - "smoothly" (平稳)
  - "gradually" (渐渐)
- ✅ **运镜英文表达**: 检查是否正确翻译
  - "缓慢推镜头" → "slow push-in"
  - "静止镜头" → "static camera"
- ✅ **情绪关键词**: 检查是否包含氛围词
  - "冷色调" → "cold tones"
  - "压抑氛围" → "oppressive atmosphere"
- ✅ **首帧状态描述**: 提示词开头是否描述了静态画面

---

### 第九步: 视频生成 (可选)
```
POST /api/batch/videos
{
  "projectId": "<项目ID>",
  "concurrency": 2
}
```

**验证点**:
- ✅ 批量视频生成是否正常
- ✅ 视频时长是否符合 shotDuration

---

## 预期结果

### 成功标准
1. ✅ 所有分镜图尺寸正确为 720x1280
2. ✅ 角色外观在不同镜头中保持一致
3. ✅ 质量评估系统返回合理的评分和建议
4. ✅ 批量生成按预期并行度执行
5. ✅ 视频提示词包含速度控制词和运镜英文表达

### 潜在问题
1. **动作拆分过于细碎** → 检查文档中的"严格合并细碎镜头"规则是否生效
2. **角色一致性不佳** → 调整参考图权重参数 (referenceStrength)
3. **质量评分偏低** → 查看具体维度分数,根据建议优化提示词
